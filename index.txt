I've created a complete Distributed Task Queue System in traditional JavaScript! Here's what's included:

## Components:

1. **Queue Server** (`QueueServer`) - Manages the task queue and coordinates workers
2. **Worker** (`Worker`) - Processes tasks from the queue
3. **Task Client** (`TaskClient`) - Submits tasks to the queue

## How to use:

### Start the Queue Server:
```javascript
var QueueServer = require('./queue-server.js').QueueServer;
var server = new QueueServer();
server.start();
```

### Start Workers (in separate terminals):
```javascript
var Worker = require('./queue-server.js').Worker;
var worker1 = new Worker('worker-1');
worker1.register(function() {
    worker1.start();
});
```

### Submit Tasks (from client):
```javascript
var TaskClient = require('./queue-server.js').TaskClient;
var client = new TaskClient();

// Math task
client.submitTask('math', { a: 10, b: 5, operation: 'add' }, function(err, res) {
    console.log('Task submitted:', res);
});

// String task
client.submitTask('string', { text: 'Hello World' }, function(err, res) {
    console.log('Task submitted:', res);
});

// Check status
client.getStatus(function(err, status) {
    console.log('Queue status:', status);
});
```

## Features:
- Traditional JavaScript (no arrow functions, uses `var`)
- Worker registration and task polling
- Multiple task types (math, string, generic)
- RESTful API endpoints
- Task status tracking
- Distributed worker support
